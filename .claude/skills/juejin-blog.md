# Skill: juejin-blog

编写掘金风格的技术博客，介绍 CesiumLite 项目的新功能模块插件。

## 使用场景

当需要为 CesiumLite 项目的新功能模块编写技术博客文章时使用此 skill。博客将发布到掘金平台，用于推广和介绍项目功能。

## 使用方式

```bash
/juejin-blog <功能模块名称>
```

例如：
- `/juejin-blog 图层切换器模块`
- `/juejin-blog 地形裁剪功能`
- `/juejin-blog 三维分析工具`

## 工作流程

1. **信息收集阶段**
   - 询问用户功能模块的基本信息（模块名称、核心功能、使用场景）
   - 询问模块所在的源码路径（如 `src/xxx/xxx.js`）
   - 询问示例页面路径（如 `examples/xxx.html`）
   - 询问是否已有相关文档或注释

2. **代码分析阶段**
   - 阅读模块源码，理解核心实现
   - 分析模块的 API 设计和使用方式
   - 提取关键代码片段作为示例
   - 对比 Cesium 原生 API 使用方式

3. **内容创作阶段**
   - 按照掘金博客结构模板编写文章
   - 突出痛点 → 解决方案 → 实战应用的思路
   - 添加代码示例和效果演示
   - 适度使用 emoji（仅标题和重要章节）

4. **文档保存阶段**
   - 将文章保存到 `docs/juejin/` 目录
   - 文件命名格式：`CesiumLite-{模块名称}介绍.md`

## 写作要点

### 1. 标题设计
- 突出痛点和解决方案
- 制造悬念，吸引点击
- 包含项目名称 "CesiumLite"

### 2. 内容结构
- **问题导向**：先讲痛点，再讲解决方案
- **对比明显**：传统方式 vs CesiumLite 方式
- **循序渐进**：基础用法 → 高级用法 → 最佳实践
- **图文并茂**：代码示例 + 架构图 + 对比表格

### 3. 代码示例
- 传统方式的代码要完整、真实、复杂
- CesiumLite 的代码要简洁、清晰、优雅
- 添加详细的注释说明
- 使用实际可运行的代码

### 4. 语言风格
- 通俗易懂，避免过度技术化
- 使用口语化表达，贴近读者
- 适当使用比喻和类比
- 保持积极、鼓励的语气

### 5. emoji 使用规范
**重要：减少 emoji 使用，避免"AI味"过重**
- 仅在文章标题使用一个 emoji
- 正文中避免使用 emoji 作为装饰
- 列表项前不使用 emoji，保持简洁专业
- 对比表格中可以使用 ✅/❌ 表示支持/不支持
- 避免在章节标题中使用 emoji（除了极少数核心章节）

### 6. 痛点描述技巧
- 使用具体的代码示例
- 列出明确的问题点
- 引起读者共鸣
- 量化复杂度（如行数、步骤数）

### 7. 解决方案展示
- 先讲设计理念
- 再讲核心实现
- 最后讲使用方式
- 突出"简单"和"优雅"

### 8. 对比效果
- 使用表格对比更直观
- 使用百分比量化改进
- 使用前后代码对比
- 突出开发效率提升

## 注意事项

1. **保持客观**：不夸大效果，实事求是
2. **代码准确**：确保示例代码可运行
3. **链接有效**：检查所有链接是否可访问
4. **排版美观**：合理使用空行和缩进
5. **技术深度**：既要通俗易懂，也要有技术深度
6. **原创性**：避免直接复制粘贴，要有自己的思考
7. **实用性**：提供可直接应用的代码和方案

# 需求分析文档

本目录存放 CesiumLite 功能插件的需求分析文档。

## 使用方法

使用 Claude Code 的自定义 skill 生成需求分析文档：

```bash
/plugin-analysis <功能插件名称>
```

例如：
```bash
/plugin-analysis 地形裁剪功能
/plugin-analysis 三维测量工具
/plugin-analysis 图层切换器
```

AI 将引导你完成交互式需求分析，并自动生成结构化的需求文档。

## 文档命名规范

需求分析文档的命名格式：

```
[插件名称]-需求分析.md
```

例如：
- `地形裁剪-需求分析.md`
- `三维测量工具-需求分析.md`
- `图层切换器-需求分析.md`

## 文档结构

每份需求分析文档包含以下章节：

1. **功能概述** - 基本信息、功能简介、目标用户、核心价值
2. **应用场景** - 典型使用场景描述
3. **功能设计** - 功能范围、对标分析
4. **技术方案** - 技术可行性、实现思路、资源管理
5. **API 设计** - 类定义、配置选项、公共方法、事件回调
6. **示例页面设计** - 演示功能、UI 设计、测试数据
7. **开发规划** - 开发步骤、工作量评估、风险挑战
8. **验收标准** - 功能验收、代码质量、文档完整性

## 文档用途

需求分析文档用于：

- ✅ 指导功能插件的开发工作
- ✅ 作为技术评审的依据
- ✅ 团队协作和知识共享
- ✅ 后续维护和迭代参考
- ✅ 生成技术博客的素材来源

## 相关 Skills

## 注意事项

1. 需求分析应在开发前完成
2. 文档应保持更新，反映实际开发情况
3. 重要的技术决策应在文档中记录
4. 使用中文编写，保持专业和规范
